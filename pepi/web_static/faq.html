Frequently Asked Questions

Pepi Web UI FAQ for version 2.2.3.dev1

Privacy & Data Collection

Does Pepi send my MongoDB logs to any external servers?

No, Absolutely Not!

Pepi operates 100% locally on your machine. Your MongoDB log files never leave your computer. All analysis, parsing, and recommendations happen entirely on your local system.

What data does Pepi collect or store?

Pepi stores data only on your local machine:

  • Uploaded Files: Files uploaded in the Web UI are written to a local temporary directory.
  • Ingest Data: When large-file ingest is started, indexed analysis data is stored locally.
  • Port Markers: Local runtime marker files are used for Web UI coordination and cleaned automatically.

No telemetry, no analytics, no external communication whatsoever.

Can I use Pepi offline?

Core analysis runs fully local. Once installed, parsing and analysis are local-only.

The Web UI references CDN-hosted frontend libraries by default, so an internet connection may be needed unless those assets are provided locally in your environment.

The only mandatory internet usage is package installation/update time (for example, pip install).

Performance & Large Files

Can I analyze very large log files (for example around 1GB)?

Yes, with guardrails. Pepi supports large files and adds safety checks before heavy analysis.

  • Preflight tiers: file size is evaluated as warn/confirm/block before proceeding.
  • UI guidance: if a file is too large, Pepi recommends trimming around the exact time window you need.
  • Streaming upload: upload processing is chunked to reduce memory pressure.
  • Deferred expensive work: some metadata (like line counts) may be computed lazily to keep uploads responsive.
What is the recommended workflow when a file is too large?

Best practice is to trim first, then analyze:

  • Use Trim Log File in Basic Info to isolate the target period.
  • Re-upload or analyze the trimmed output for better responsiveness and lower resource usage.
  • Use sampling and narrow filters when exploring large windows.

Analysis Modes

What is the difference between raw analysis and ingest-backed analysis?

Pepi supports both read paths in the current API:

  • Raw: reads directly from the uploaded file.
  • Ingest: reads from locally indexed data generated by a background ingest job.

For very large logs, ingest-backed queries can improve repeated analysis and responsiveness.

How do I check ingest progress?

Ingest jobs expose status and cancellation endpoints that the UI can poll:

  • /api/ingest/{file_id}/start
  • /api/ingest/{file_id}/status
  • /api/ingest/{file_id}/cancel

TMP And Troubleshooting

Where are uploaded files written and how can I change it?

Upload temporary directory resolution order is:

  • PEPI_UPLOAD_TMPDIR
  • TMPDIR
  • system temp directory

Disk guardrails can be tuned with PEPI_UPLOAD_MIN_FREE_MB and PEPI_UPLOAD_HEADROOM_FACTOR.

What does a 507 upload error mean?

507 indicates insufficient storage in the temporary upload location.

  • Free disk space, or
  • set PEPI_UPLOAD_TMPDIR to a larger partition, or
  • trim the log to a smaller time range and retry.

The UI surfaces backend details/hints, and system temp health is available via /api/system/tmp-health.

Why are terminal logs quieter in Web UI mode?

By default, Web UI runtime output is reduced to focus on errors/warnings.

Request-level HTTP logging can be enabled by setting PEPI_HTTP_LOG=true.
