Frequently Asked Questions

Everything you need to know about Pepi

Privacy & Data Collection

Does Pepi send my MongoDB logs to any external servers?

No, Absolutely Not!

Pepi operates 100% locally on your machine. Your MongoDB log files never leave your computer. All analysis, parsing, and recommendations happen entirely on your local system.

What data does Pepi collect or store?

Pepi stores data only on your local machine:

  • Cached Analysis: Parsed log data is cached locally for 7 days to speed up repeated analyses
  • Uploaded Files: Log files you upload via the web UI are temporarily stored in your system's temp directory

No telemetry, no analytics, no external communication whatsoever.

Can I use Pepi offline?

Yes! Once installed, Pepi works completely offline. The web UI loads from your local system, and all processing happens locally. The only time you need internet is:

  • Initial installation: pip install

Performance & Resources

Can I analyze large log files (GB+)?

Yes! Pepi handles large files efficiently:

  • Streaming parser: Processes files line-by-line (doesn't load entire file into memory)
  • 7-day cache: Re-analyzing the same file is instant
  • Progress bars: Shows real-time progress with tqdm
  • Data sampling: Time series automatically samples large datasets (>10k points) for browser performance

Technical Details

What MongoDB versions are supported?

Pepi supports MongoDB 4.4+ (versions that use the JSON log output format introduced in MongoDB 4.4). It has been tested with MongoDB 4.4, 5.x, 6.x, and 7.x.

MongoDB 4.4 introduced structured JSON logging (Relaxed Extended JSON v2.0) for all log destinations, which Pepi requires for accurate parsing.

Where is the cache stored?

Cache is stored in ~/.pepi_cache/ with 7-day TTL. You can clear it anytime with:

pepi --clear-cache
