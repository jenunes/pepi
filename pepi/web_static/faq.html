Frequently Asked Questions

Pepi Web UI FAQ for version 2.2.5

Privacy & Data Handling

Does Pepi send my MongoDB logs to any external servers?

No, Absolutely Not!

Pepi operates 100% locally on your machine. Your MongoDB log files never leave your computer. All analysis, parsing, and recommendations happen entirely on your local system.

What data does Pepi collect or store?

Pepi stores data only on your local machine:

  • Uploaded Files: Files uploaded in the Web UI are written to a local temporary directory.
  • Ingest Data: When large-file ingest is started, indexed analysis data is stored locally.
  • Port Markers: Local runtime marker files are used for Web UI coordination and cleaned automatically.

No Pepi log telemetry. The CLI may query GitHub for version tags in the background; the browser loads CDN assets for the UI.

Can I use Pepi offline?

Core analysis runs fully local. Once installed, parsing and analysis are local-only.

The Web UI references CDN-hosted frontend libraries by default, so an internet connection may be needed unless those assets are provided locally in your environment.

The only mandatory internet usage is package installation/update time (for example, pip install).

Performance & Large Files

Can I analyze very large log files (for example around 1GB)?

Yes, with guardrails. Pepi supports large files and adds safety checks before heavy analysis.

  • Preflight tiers: file size is evaluated as warn/confirm/block before proceeding.
  • UI guidance: if a file is too large, Pepi recommends trimming around the exact time window you need.
  • Streaming upload: upload processing is chunked to reduce memory pressure.
  • Deferred expensive work: some metadata (like line counts) may be computed lazily to keep uploads responsive.
What is the recommended workflow when a file is too large?

Best practice is to trim first, then analyze:

  • Use Trim Log File in Basic Info to isolate the target period.
  • Re-upload or analyze the trimmed output for better responsiveness and lower resource usage.
  • Use sampling and narrow filters when exploring large windows.
How does line sampling work?

With sample=100 and more than 50k lines, parsers use automatic stride (every 5th, 10th, or 20th line by size). Lower sample forces coarser strides. Time Series plots also downsample client-side (2000 or 6000 points depending on large-dataset mode at ≥200k lines).

Raw vs Ingest Behavior

What is the difference between raw and ingest?

Raw reads the uploaded log file on each heavy request. Ingest runs a background job that inserts rows into local SQLite; source=ingest reads that DB. The UI switches to ingest when the latest job status is completed.

How do I check ingest progress?

Poll GET /api/ingest/{file_id}/status. Start with POST /api/ingest/{file_id}/start (optional query force=true). Cancel with POST /api/ingest/{file_id}/cancel.

If the log file changes, does ingest refresh?

No. Re-run ingest. Raw mode always reads the path currently registered for that upload.

Query Diagnostics & Index Recommendations

How are patterns identified?

parse_queries groups COMMAND log lines by namespace, operation, and normalized pattern. POST /api/analyze/{file_id}/query-diagnostics needs an exact key from the last queries run (same filters and sample).

How are index recommendations sourced?

POST /api/analyze/{file_id}/index-recommendations: with raw_log_line in JSON, recommendation_source is selected_example; with pattern stats only, selected_pattern_fallback; otherwise bulk_all_patterns (capped by top_n). Responses set has_llm: false (rule-based).

Why are query examples empty?

Examples scan the log for up to five COMMAND lines matching the pattern. Sampling or mismatched pattern strings can yield zero examples even when aggregate stats exist.

Troubleshooting

Stale or empty results

CLI: pepi --fetch /path/to/existing.log --clear-cache clears all pickles under ~/.pepi_cache. UI: delete file and re-upload, or re-run ingest. Corrupt pickles delete themselves on read failure.

404 on query diagnostics

The (namespace, operation, pattern) key is missing from aggregated stats—re-run Analyze Queries with matching filters and sample.

CLI vs Web numbers differ

Compare sampling percentage, raw vs ingest source, and filters. CLI --sort-by for connections only reorders for opened and closed.

Storage & TMP Errors

Where are uploaded files written and how can I change it?

Upload temporary directory resolution order is:

  • PEPI_UPLOAD_TMPDIR
  • TMPDIR
  • system temp directory

Disk guardrails can be tuned with PEPI_UPLOAD_MIN_FREE_MB and PEPI_UPLOAD_HEADROOM_FACTOR.

What does a 507 upload error mean?

507 indicates insufficient storage in the temporary upload location.

  • Free disk space, or
  • set PEPI_UPLOAD_TMPDIR to a larger partition, or
  • trim the log to a smaller time range and retry.

The UI surfaces backend details/hints, and system temp health is available via /api/system/tmp-health.

Why are terminal logs quieter in Web UI mode?

By default, Web UI runtime output is reduced to focus on errors/warnings.

Request-level HTTP logging can be enabled by setting PEPI_HTTP_LOG=true.

Common Operational Tips

Multiple servers on one machine

Up to three pepi.web_api processes may listen on ports 8000–8002. Stop one before starting another.

Trim first

Use Basic Info trim or CLI --trim to shrink the window before deep analysis.
